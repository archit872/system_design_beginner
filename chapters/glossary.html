REFERENCE

Glossary

A quick reference for **system design vocabulary**. Each entry has a plain definition, a formal note, and a one-sentence example.

A–C

API (Application Programming Interface)
Plain: A set of rules for how software pieces talk. Formal: Defined request/response contract between components. Example: The Twitter API lets you fetch tweets with `GET /tweets`.
Availability
Plain: How often the system is up. Formal: Percentage of time a system meets service goals (often “nines”). Example: 99.9% availability ≈ 8h downtime/year.
Backup
Plain: Copy of data for recovery. Formal: Stored snapshot for disaster recovery or corruption rollback. Example: Daily S3 snapshots of a DB.
Bottleneck
Plain: Slowest part that limits performance. Formal: Component with least capacity relative to demand. Example: DB write lock slows checkout throughput.
Cache
Plain: Fast storage for hot data. Formal: Memory layer reducing latency/cost at expense of consistency. Example: Redis stores recent timeline queries.
Circuit breaker
Plain: Switch to cut traffic to a failing service. Formal: Pattern that halts calls to unhealthy dependency and returns fallback. Example: Disable recommendations API when it times out repeatedly.

D–L

Database
Plain: Organized data store. Formal: Structured collection with query language, guarantees (ACID/eventual). Example: PostgreSQL stores user accounts.
Eventual consistency
Plain: Data copies may differ briefly but converge. Formal: Guarantee that given no updates, all replicas converge to same value. Example: After a post, followers may see it seconds later.
Failover
Plain: Automatic switch to backup. Formal: Mechanism that reroutes requests when a component fails. Example: DB replica takes over if primary crashes.
Graceful degradation
Plain: System still works partially when broken. Formal: Ability to maintain limited service under component failure. Example: Show cached feed if live updates fail.
Idempotency
Plain: Repeat request safely. Formal: Property where repeated operations yield same result. Example: Retrying payment POST doesn’t double charge.
Latency
Plain: Time delay per request. Formal: Duration from request to response, measured in ms. Example: Webpage loads in 120 ms.
Load
Plain: Amount of work hitting system. Formal: Incoming requests per second and resource demand. Example: Login endpoint at 500 RPS.
Load balancer
Plain: Traffic cop for servers. Formal: Middleware distributing traffic among instances based on policy and health checks. Example: HAProxy spreads requests across 4 web servers.

M–R

Microservices
Plain: Many small services instead of one app. Formal: Architectural style decomposing apps into loosely coupled services. Example: Auth, payments, and orders run separately.
Monitoring
Plain: Watching system health. Formal: Continuous collection of metrics (latency, errors, saturation). Example: Grafana dashboard for API latency.
NoSQL
Plain: Non-relational database family. Formal: Key-value, document, wide-column, or graph DBs trading schema for flexibility. Example: MongoDB stores JSON documents.
Queue
Plain: Line for tasks. Formal: Data structure/process holding tasks FIFO for async workers. Example: RabbitMQ stores image resize jobs.
Redundancy
Plain: Having spares. Formal: Extra components so one can fail without service loss. Example: Two app servers in different zones.
Replication
Plain: Copying data. Formal: Maintaining multiple synchronized instances of data for availability and scale. Example: Primary DB replicates to read replicas.

S–Z

Scalability
Plain: Ability to handle more load. Formal: Property of a system to increase capacity while meeting SLOs. Example: Add servers behind LB to double throughput.
SLI (Service Level Indicator)
Plain: Measurement of performance. Formal: Quantitative measure of service quality (latency, error rate). Example: 99% of logins under 200 ms.
SLO (Service Level Objective)
Plain: Target goal for reliability. Formal: Agreement on acceptable SLI thresholds. Example: 99.9% availability per quarter.
SQL
Plain: Relational database language. Formal: Structured Query Language used in relational DBs with ACID guarantees. Example: `SELECT * FROM users WHERE id=7`.
Stateful
Plain: Keeps memory between requests. Formal: Service retains user/session state across calls. Example: Chat room server holding active participants in memory.
Stateless
Plain: No memory between requests. Formal: Each request independent; any server can serve it. Example: Login API verifying JWT token per call.
Throughput
Plain: How much work per time. Formal: Number of requests/transactions processed per second. Example: 5k RPS sustained on checkout.
