Reference Glossary

Glossary

Each entry lists a plain definition, a formal definition, a common pitfall, and a short example.

A

API
Plain: How programs talk. Formal: A contract (endpoints/messages) between components over a protocol. Pitfall: Designing APIs without error handling or versioning. Example: POST /links returns 201 with a short code.
Availability
Plain: Chance the service is up. Formal: Fraction of requests within SLO over time. Pitfall: Confusing with durability. Example: 99.9% monthly availability target.

B

Backpressure
Plain: Telling senders to slow down. Formal: Flow-control mechanisms that bound queue growth and resource use. Pitfall: Unlimited buffers. Example: WebSocket server caps messages/sec per connection.
Bulkhead
Plain: Isolation wall. Formal: Resource isolation (pools/threads) to contain failures. Pitfall: One pool for all dependencies. Example: Separate DB and cache connection pools.

C

Cache
Plain: Save work to reuse later. Formal: A faster tier storing results with defined freshness. Pitfall: No invalidation plan. Example: Redis cache-aside with 60 s TTL + jitter.
CDN
Plain: Servers near users. Formal: Geo-distributed edge network that caches content and terminates TLS. Pitfall: Caching personalized pages. Example: Cache 301 redirects for hot short links.
Circuit Breaker
Plain: Stop calling what’s failing. Formal: Control that opens on high error/latency, then probes recovery. Example: Pause calls to a slow dependency and serve fallbacks.
Consistency
Plain: How fresh/ordered reads are. Formal: Guarantees on visibility of writes (strong, eventual, causal). Pitfall: Overpromising strong everywhere. Example: Eventual consistency for analytics counts.

D

Durability
Plain: Data won’t vanish. Formal: Probability stored data survives faults. Pitfall: Untested backups. Example: Nightly encrypted snapshots; monthly restore drills.

I

Idempotency
Plain: Safe to retry. Formal: Replays yield equivalent state. Pitfall: Missing request IDs. Example: Idempotency-Key for POST to avoid duplicate links.
Index
Plain: Fast lookup guide. Formal: Data structure (B-tree/hash) that accelerates filters/sorts. Pitfall: Non-sargable filters. Example: messages(conversation_id, created_at DESC).

L

Latency
Plain: Time to respond. Formal: End-to-end delay (p50/p95/p99). Pitfall: Reporting averages only. Example: p95 redirect < 120 ms globally.
Load Balancer
Plain: Traffic router. Formal: L4/L7 component that distributes requests across replicas. Pitfall: No health checks/draining. Example: Least-connections during flash sale.

R

Rate Limiting
Plain: Caps on requests. Formal: Algorithms (token bucket/leaky bucket) to bound RPS. Pitfall: Global limits only. Example: Per-IP and per-key limits on POST /links.
Replication
Plain: Extra copies. Formal: Leader–follower or quorum models for durability/availability. Pitfall: Ignoring lag. Example: Read-your-writes stickiness after user updates.
RPO / RTO
Plain: Data loss / downtime budgets. Formal: Recovery Point/Time Objectives. Pitfall: Undefined targets. Example: RPO ≤ 1 min, RTO ≤ 30 min with warm standby.

S

Scalability
Plain: Handle more work. Formal: Throughput increases with resources while meeting SLOs. Pitfall: Scale out without measuring bottlenecks. Example: Add replicas and a cache to keep p95 < 300 ms.
SLI / SLO
Plain: Metric / target. Formal: Indicators and objectives governing reliability. Pitfall: Alerting on internals only. Example: p95 < 250 ms, 99.9% availability.

T

Throughput
Plain: Work per second. Formal: Completed requests/time unit. Pitfall: Chasing RPS while p95 spikes. Example: 1.5k RPS/replica at 60% CPU.
TTL
Plain: Expiration time. Formal: Maximum cache lifetime before refresh. Pitfall: Synchronized expiries. Example: 60–120 s TTL with ±20% jitter.
